# 买衣服决策 Skill

目标：用最少的时间，买到穿着满意、不后悔的衣服。

---

## 第一步：需求澄清（防止买错的关键）

用户说想买衣服时，先问清楚以下信息（可以一次问，不要分多轮）：

```
1. 场景：主要在哪穿？（上班/日常外出/运动/约会/居家/特殊场合）
2. 缺口：衣柜里现在缺什么？（某个场景没有合适的，还是某件穿坏了要替换）
3. 预算：单件大概多少？（不用精确，给个范围就行）
4. 限制：有没有不要的颜色/款式/材质？（比如不穿白色、不要纯棉、不喜欢宽松）
5. 体型参考：身高体重，或者常穿的尺码（S/M/L 或具体数字）
```

如果用户信息已经足够清晰，跳过已知项直接进入第二步。

---

## 第二步：建立选购标准（在搜索前先定标准）

根据用户的场景和需求，输出一张"选购标准卡"：

```
## 这次买衣服的标准

**场景：** [用户的场景]
**必须满足：**
- [ ] [标准1，例如：适合办公室，不过于休闲]
- [ ] [标准2，例如：深色系，耐脏]
- [ ] [标准3，例如：价格在300元以内]

**加分项：**
- 有口袋
- 免烫材质
- 可机洗

**一票否决：**
- 白色
- 需要干洗
- 过于显瘦/宽松的版型
```

**在这张卡确认之前，不建议开始搜索。** 让用户确认或修改标准。

---

## 第三步：推荐逻辑（最多推荐3件）

用标准筛选后，**最多推荐3件**，每件给出完整评估：

```
### 推荐 1：[品牌] [商品名]
- 链接：[URL]
- 价格：[价格]
- 为什么推荐：[用1-2句话说明它符合哪几条标准]
- 潜在风险：[可能不满意的点，提前说清楚]
- 适合人群：[什么体型/风格的人更适合]
```

**推荐原则：**
- 宁缺毋滥，不够好就不推
- 必须说清楚"潜在风险"，避免用户收到后失望
- 优先推荐有充足评价（100+）、退换货方便的商品
- 避免推荐"看图很好看但评价两极分化"的商品

---

## 第四步：防退款清单（下单前必看）

在用户决定购买前，给出以下提示：

```
## 下单前确认清单

- [ ] 你看的是真实买家图，不是模特图（点开评价里的买家秀）
- [ ] 尺码你确认过了（对照了品牌尺码表，不是凭感觉选的）
- [ ] 材质你可以接受（看了吊牌成分，比如是否纯棉/聚酯纤维）
- [ ] 退换货政策确认（是否支持7天无理由，运费谁出）
- [ ] 你已经想象过它搭配什么穿（不是单独看好看，是整体穿搭能用上）
```

只有清单全部确认，才建议下单。

---

## 第五步：搭配验证（可选）

如果用户不确定新衣服能不能跟已有的搭，问：

> "你现在有什么颜色的下装/上装/鞋子？"

然后给出具体搭配建议：
- 这件上衣 + [具体裤子颜色款式] + [鞋子] = 什么风格
- 如果衣柜里没有合适的配搭，提前说，避免买了穿不出去

---

## 注意事项

- **不要推荐超出预算的商品**，除非有充分理由且明确说明
- **不要因为"便宜"就推荐**，便宜不合适依然是浪费钱
- **退款成本不只是钱**，还有时间和精力，帮用户在买之前就排除风险
- 如果用户信息不足以给出好推荐，宁可多问一轮，也不要随便推
