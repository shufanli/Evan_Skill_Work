# MECE 结构化问题分析

用户提出一个问题后，按以下流程进行结构化拆解。

---

## 第一步：明确问题

把用户的问题重新表述为一句清晰、具体、可拆解的问题。如果问题模糊，先追问澄清。

输出格式：
```
**待分析的问题：** [重新表述的问题]
```

---

## 第二步：选择切分方法

从以下 4 种方法中选择最适合的 1-2 种开始切分：

| 方法 | 适用场景 | 示例 |
|------|---------|------|
| **公式法** | 问题背后有数学/财务公式 | 利润=收入-成本 → 开源 or 节流 |
| **子目录列举法** | 可穷举子分类，借用成熟框架 | 增长分析 → AARRR（获客/激活/留存/收入/推荐） |
| **流程法** | 问题和时间/阶段/步骤有关 | 用户旅程 → 认知→考虑→购买→使用→复购 |
| **逻辑框架法** | 问题范围模糊，需要先粗切 | 内部 vs 外部、短期 vs 长期、可控 vs 不可控 |

输出格式：
```
**选用方法：** [方法名]
**理由：** [为什么选这个方法]
```

---

## 第三步：逐层切分（至少三层）

严格遵循 MECE 原则进行切分：
- **ME（不重叠）**：每个分支之间相互独立，没有交叉
- **CE（不遗漏）**：所有分支加起来穷尽全部可能

用树状结构输出，**必须切到第三层**：

```
问题
├── 第一层分支 A
│     ├── 第二层分支 A1
│     │     ├── 第三层 A1a
│     │     └── 第三层 A1b
│     └── 第二层分支 A2
│           ├── 第三层 A2a
│           └── 第三层 A2b
└── 第一层分支 B
      ├── 第二层分支 B1
      │     ├── 第三层 B1a
      │     └── 第三层 B1b
      └── 第二层分支 B2
```

每层切完后自检：
- [ ] 各分支是否独立不重叠？
- [ ] 所有分支加起来是否穷尽？
- [ ] 维度是否具体可衡量？

---

## 第四步：借用成熟框架（如适用）

如果问题涉及以下领域，主动借用对应的成熟框架：

**市场与竞争：** 4P / 波特五力 / 3C / STP
**企业内部：** 价值链 / 7S / 人-系统-流程
**战略决策：** SWOT / BCG矩阵 / 安索夫矩阵
**用户与产品：** 用户旅程 / AARRR / Kano模型
**宏观环境：** PEST / PEMCT

标注使用了哪个框架，并说明为什么适用。

---

## 第五步：生成多维图谱（如适用）

如果问题适合用两个维度交叉分析，生成四象限图谱：

```
            高 [Y轴维度]
            │
   [象限名]  │  [象限名]
   策略：xxx │  策略：xxx
            │
  ──────────┼──────────
            │
   [象限名]  │  [象限名]
   策略：xxx │  策略：xxx
            │
            低          高 [X轴维度]
```

每个象限给出：名称 + 一句话策略建议。

---

## 第六步：输出结论与行动建议

```
## 分析结论

**核心发现：**
1. [最重要的发现]
2. [第二重要的发现]
3. [第三重要的发现]

**建议行动：**
- [ ] [具体可执行的行动 1]
- [ ] [具体可执行的行动 2]
- [ ] [具体可执行的行动 3]

**需要进一步验证的假设：**
- [假设 1]：需要什么数据来验证？
- [假设 2]：需要什么数据来验证？
```

---

## 注意事项

- 如果用户提供的信息不足，主动追问关键信息再开始分析
- 切分过程中发现某个分支特别重要，可以重点深挖到第四、五层
- 鼓励用户对分析结果提出质疑和补充
- 分析完成后提醒用户："这个分析可以从其他维度再切一次，要不要换个角度看？"（3-3原则：至少3棵逻辑树，每棵至少3层）
